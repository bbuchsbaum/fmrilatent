# fmrilatent

## Purpose

R package for latent space representations of fMRI neuroimaging data. Provides S4 classes (`LatentNeuroVec`) for memory-efficient storage and manipulation of dimensionality-reduced brain data using matrix factorization (basis x loadings). Integrates with the `neuroim2` package for standard neuroimaging data structures.

The core idea: represent 4D fMRI data as `X = B * L' + c` where B is a temporal basis, L is spatial loadings, and c is a voxel-wise offset. This enables efficient PCA, ICA, wavelet, Slepian, HRBF, and other decompositions.

## For AI Agents

### Working In This Directory

- Always run `Rscript -e "devtools::load_all()"` after modifying R source files
- Run `Rscript -e "devtools::document()"` after changing roxygen2 comments (regenerates `man/` and `NAMESPACE`)
- Collation order in `DESCRIPTION` matters — `latent_handles.R` must load before `all_class.R`
- 2-space indent, snake_case for functions/objects
- S4 methods use `setMethod("generic", "Class", ...)` pattern
- roxygen2 comments for user-facing functions; use `@keywords internal` for private helpers
- Keep files ASCII; prefer pure functions; avoid side effects or hidden globals

### Testing Requirements

- Run `Rscript -e "devtools::test()"` before committing
- Full check: `Rscript -e "devtools::check()"`
- Tests use testthat edition 3: `tests/testthat/test-*.R`
- Use `set.seed()` for reproducible random data in tests

### Common Patterns

- **Handle pattern**: `BasisHandle`/`LoadingsHandle` defer materialization — `basis(x)` calls `materialize_basis()` only when needed
- **Spec pattern**: `spec_time_*()` / `spec_space_*()` create lightweight descriptors; `encode_spec()` dispatches to builders
- **Lift pattern**: `lift(reduction, spec)` computes bases on coarsened graphs, lifts to ambient space
- **Registry**: Shared cache for materialized dictionaries, controlled by `options(fmrilatent.registry.enabled = TRUE/FALSE)`
